# Feng Farm 项目管理规范

## 团队角色

| 角色 | 代号 | 职责 |
|------|------|------|
| 项目经理 | PM | 统筹、协调、验收 |
| 策划 | PD | 需求、验收标准、策划验收 |
| 程序 | DEV | 技术实现、自测、修复 |
| 测试 | QA | 测试用例、测试执行、报告 |

## 工作流程

```
需求(PD) → PM审核 → 开发(DEV) → 提测 → 测试(QA) → 策划验收(PD) → PM验收 → 完成
```

## 状态流转

- 待评审 → 开发中 → 提测 → 测试中 → 策划验收 → PM验收 → 已完成
- 任何环节可打回：测试中→开发中, 策划验收→开发中, PM验收→开发中

## 文档规范

1. **需求文档**：唯一ID + 验收标准 + 优先级
2. **技术文档**：架构 + API + 部署
3. **测试文档**：计划 + 用例 + 报告

## 交付物清单

| 阶段 | 交付物 | 责任人 |
|------|--------|--------|
| 需求 | 《需求文档》+《验收标准》 | PD |
| 开发 | 代码 + 《技术文档》 | DEV |
| 测试 | 《测试报告》+ Bug清单 | QA |
| 验收 | 签字确认 | PD + PM |
