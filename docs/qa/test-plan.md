## 1. 测试目标

### 1.1 总体目标
确保 Feng Farm 多人协作种田游戏的核心功能稳定可用，多人实时同步流畅，用户体验符合GDD设计预期。

### 1.2 具体目标
- 验证核心玩法循环（种植→浇水→收获）完整可用
- 验证多人实时协作功能正常
- 验证游戏性能达到验收标准（延迟<100ms）
- 发现并记录关键缺陷

### 2.1 测试范围（In Scope）

## 6. 验收标准

### 6.1 功能验收标准

| 检查项 | 标准 | 优先级 |
|--------|------|--------|
| 新手引导 | 玩家3分钟内学会基础操作 | P0 |
| 核心循环 | 种植→浇水→收获的循环流畅 | P0 |
| 多人同步 | 延迟<100ms，无卡顿 | P0 |
| 视觉反馈 | 每次操作都有明确反馈 | P0 |

### 6.2 性能验收标准

| 指标 | 目标值 | 测试方法 |
|------|--------|----------|
| 首次加载时间 | < 3秒 | Lighthouse |
| WebSocket延迟 | < 100ms | Ping测试 |
| 同房间人数上限 | 20人 | 压测 |
| 帧率 | > 30fps | Chrome DevTools |

### 6.3 缺陷验收标准

| 严重程度 | 数量限制 | 说明 |
|----------|----------|------|
| 致命 (Critical) | 0 | 系统崩溃、数据丢失 |
| 严重 (Major) | 0 | 主要功能不可用 |
| 一般 (Minor) | ≤ 5 | 次要功能问题 |
| 轻微 (Trivial) | ≤ 10 | UI/UX小问题 |

| 移动端兼容性 | 中 | 中 | 明确本期不覆盖移动端 |
